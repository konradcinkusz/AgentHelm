AgentHelm — Architecture

Jak działa Copilot CLI w AgentHelm?

Skąd musi być uruchomiony Bridge, dlaczego ścieżka musi istnieć lokalnie, i co można zmienić.

dowolne miejsce
HTTP / SSE — POST /api/sessions, GET /api/sessions/{id}/stream

Dlaczego Bridge musi być lokalny

⚠️
Kod źródłowy (docker-compose.yml, linia 6–8) wprost ostrzega:
„ACP agents (Copilot CLI, Claude Code, Gemini) are NOT available inside containers — they need your local environment and auth."
PowódSzczegół
auth Lokalne dane logowania Copilot CLI czyta token GitHub z lokalnego keychain / ~/.config/gh. Kontener tego nie ma.
process Child process Process.Start(psi) w ProcessTransport — Copilot startuje jako dziecko Bridge na tej samej maszynie.
fs Walidacja ścieżki Directory.Exists(cwd) w SessionManager.CreateAsync() — ścieżka musi istnieć na maszynie Bridge, nie na komputerze użytkownika.
security Guard path Każde żądanie fs/read / fs/write spoza cwd jest odrzucane przez AcpClient z błędem -32602.

Jak cwd trafia do agenta — dwa miejsca

// 1. OS-level — ProcessStartInfo (AcpClient.cs:29) new ProcessStartInfo { FileName = "copilot", WorkingDirectory = workingDirectory, ← musi istnieć na Bridge RedirectStandardInput = true, RedirectStandardOutput = true }; // 2. ACP protocol — session/new payload (AcpClient.cs:171) var params_ = new JsonObject { ["cwd"] = _cwd, ← ta sama ścieżka, wysłana do Copilot CLI ["mcpServers"] = new JsonArray() }; await RequestAsync("session/new", params_, ct);

Jak łączy się z nowym pickerem lokalnym

ℹ️
Nowy picker otwiera natywne okno systemu i zwraca tylko nazwę folderu (ograniczenie bezpieczeństwa przeglądarki — pełna ścieżka niedostępna). Nadal trzeba wpisać ścieżkę, pod którą ten folder jest widoczny dla Bridge.
ScenariuszLokalny picker pokazujeCo wpisać w pole

Docker + volume mount Folder myrepo na C:\Repos\myrepo /repos/myrepo (ścieżka kontenera)
Bridge na zdalnym serwerze Folder lokalny — nieprzydatny Ścieżka na serwerze, wpisana ręcznie

Czy można to zmienić — opcje

✓ działa dziś Bridge lokalnie

Uruchom Bridge przez run.ps1 / run.sh na swoim komputerze. AgentHelm.Web może być gdziekolwiek — łączy się z http://localhost:5199. Ścieżki lokalne działają w polu "Working directory".

✓ działa dziś Docker + volumes

W docker-compose.yml dodaj volumes: - C:\Repos:/repos. Copilot jest niedostępny w kontenerze, ale ścieżki montowane są widoczne dla Bridge jeśli Bridge jest lokalny.

stub CopilotSdkAdapter

Istnieje szkielet CopilotSdkAdapter.cs (gated #if COPILOT_SDK), ale ma 4× NotImplementedException. Wymagałby dodania NuGet SDK od GitHub i implementacji. Nadal lokalny.

wymaga pracy Remote Bridge

Zdalny Bridge wymagałby tunelu (SSH / ngrok) do lokalnego Copilot CLI, lub protokołu zdalnego wykonania. ACP jest zaprojektowany jako stdio lokalny — nie ma natywnego wsparcia remote.

TL;DR

✅
Copilot CLI, Bridge i repo muszą być na tej samej lokalnej maszynie. Web UI może być hostem gdziekolwiek — to tylko frontend. Nowy picker lokalny pomaga zobaczyć nazwę folderu, ale ścieżkę w polu wciąż podajesz z perspektywy Bridge, nie przeglądarki.
