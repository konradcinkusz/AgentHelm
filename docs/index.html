AgentHelm

A web cockpit for AI coding agents. One GUI to drive GitHub Copilot CLI, Claude Code, Gemini CLI — and any of the ~50 agents speaking the Agent Client Protocol — with transparent transcripts, explicit tool permissions, and an audit trail.

→ Get Started View on GitHub Tutorial

What it is

Steer the agent.
You control the tools.

AgentHelm is the missing GUI layer between you and your AI agent. Every tool call the agent wants to make — read a file, run a command, push a change — passes through your explicit approval before it happens.

Every decision — human or automatic — lands in a permanent transcript. No silent approvals. No mystery about what ran.

Sibling project of CopilotScope: Scope observes (what was that session worth?), Helm steers (run the session, approve the tools, keep the record).

Features

Everything you need to run agents safely

🔐

Permission gateway

Every tool call requires your approval. Three policies: Ask (always), Auto-read (reads only), YOLO (explicit opt-in per session). Every decision is audited.

📜

Transparent audit trail

Full transcript with tool calls inline — human approvals, auto-approvals, rejections. Persisted to Postgres with 1-second write-behind.

🔀

Multi-agent by protocol

One ACP client covers ~50 agents. Agents are configuration, not code — add any ACP-speaking command to appsettings.json.

📂

Git diff viewer

Review working-tree changes vs HEAD. Accept (git add) or Reject (git checkout HEAD) per file — both actions audited and path-guarded.

💻

Integrated terminal

xterm.js shell inside the session cwd. PTY on Unix (real interactive programs, colors), pipe on Windows. "→ Prompt" feeds output into the agent.

🔁

Session resume

Archived sessions carry the agent-side session id. One click resumes via ACP session/load — history replays into the transcript.

🤝

Agent handoff

Continue a conversation with a different agent. Helm prefills the new session with an attributed summary — never auto-sent, you review first.

📊

CopilotScope integration

Quality scores from CopilotScope inline in the session header — time-window correlation against overlapping Scope sessions.

Quick start

Up and running in under a minute

Prerequisites: .NET SDK 8.0.303+. Docker is optional (only for persistent history).

# Clone and start with .NET Aspire git clone https://github.com/konradcinkusz/AgentHelm.git cd AgentHelm dotnet run --project src/AgentHelm.AppHost

Aspire starts Postgres (container), Bridge (local process), and Web UI (local process). Open the URL Aspire prints for web. Bridge and Web run locally on purpose — ACP agents need your local environment and auth.

# Download the latest release zip from GitHub Releases # Unpack it, then: # macOS / Linux ./run.sh # Windows .\run.ps1 # Bridge → http://127.0.0.1:5199 # Web UI → http://127.0.0.1:5200

Requires only the .NET 8 ASP.NET Core runtime (not the full SDK). No Docker needed — runs without Postgres in memory-only mode.

# Pull and run from GHCR (no clone needed) curl -O https://raw.githubusercontent.com/konradcinkusz/agenthelm/master/docker-compose.ghcr.yml docker compose -f docker-compose.ghcr.yml up # UI → http://localhost:5300 # API → http://localhost:5299

Note: ACP agents (Copilot CLI, Claude Code, Gemini) cannot run inside containers — they need your local environment and auth. The built-in echo agent works out of the box. For real agents use the Aspire or zip option above.

Try it in 30 seconds — no real agent needed

  1. Click ＋ New → pick Echo (built-in demo agent)
  2. Set any existing directory as working directory → Start session
  3. Type a prompt — watch it stream back
  4. Type something with the word tool — the permission banner appears. Allow or reject.
